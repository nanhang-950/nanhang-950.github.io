整数溢出

简介

在 C 语言中，整数的基本数据类型包括 short、int、long 和 char，每种数据类型都可以是有符号（signed）或无符号（unsigned）。这些数据类型在内存中占用的大小和它们表示的值的范围是不同的。

在 《C和指针》 一书中，将char型也归类为整数类型，严格来说确实如此。虽然它通常用来表示字符，但从 C 语言的角度来看，char 实际上是整数类型的一种。其默认大小为 1 字节（8 位），而且它的值范围依赖于是否是有符号类型。char型之所以被视为字符型，仅仅是因为它使用 ASCII 码值来表示字符，但是它也可以作为整型使用。

类型 字节 范围
char 1byte (byte) -128 ~ 127（有符号）
unsigned char 1byte (byte) 0 ~ 255（无符号）
short int 2byte (word) -32768 ~ 32767（有符号）
unsigned short int 2byte (word) 0 ~ 65535（无符号）
int 4byte (dword) -2147483648 ~ 2147483647（有符号）
unsigned int 4byte (dword) 0 ~ 4294967295（无符号）
long int 8byte (qword) -9223372036854775808 ~ 9223372036854775807（有符号）
unsigned long int 8byte (qword) 0 ~ 18446744073709551615（无符号）

众所周知，两位十六进制数表示一个字节，正如上表中所示，char 类型占用一个字节，因此使用两位十六进制数表示。

溢出原理

当程序中的数据超过其数据类型的范围，则会造成溢出，整数类型的溢出被称为整数溢出。

整数溢出可以分为上界溢出和下界溢出，接下来我们分别介绍其原理。

上界溢出

查看运行结果，每个数据类型都从最大值变成了最小值，这样我们就称它产生了上界溢出。

之所以称为上界溢出，是因为它们都是因为数据类型的最大值增加 1 后，导致结果回绕到数据类型的最小值。这种溢出是由于数据类型所能表示的最大值达到了上限，从而 “溢出” 到数据类型的最小值。

这种情况的发生与计算机底层的表示方式以及数据类型的特性有关。因为计算机底层指令是不区分有符号和无符号的，数据都是以二进制形式存在（编译器的层面才对有符号和无符号进行区分，产生不同的汇编指令）。

我们用一张滴水逆向的神图来解释。

以上述代码中的有符号char型和无符号char型为例，在上述代码中有符号的char型和无符号的char的最大值分别是0x7F和0xFF，都是正数。由于都加了 1 ，所以有符号的int型由正数溢出为了负数，无符号的char型由最大值溢出为了最小值。

上界溢出有两种情况：一种是0x7F + 1，另一种是0xFF + 1。

  • 0x7F + 1 = 0x80

第一钟仅针对有符号数有作用，这是因为在无符号类型中0x80表示的是128，所以0x7f + 1就相当于127 + 1 = 128。因此这种上界溢出对无符号整型没有什么影响。

但是在有符号整型中，0x7f表示的是127，但是0x80表示的是-128。简单来说就是就是在有符号整型中127 + 1 = -128，所以0x7F + 1在有符号数中从正数溢出为了负数。

  • 0xFF + 1 = 0x00

第二种情况仅针对无符号数有用。这是因为在无符号char型中，数值的范围是从0到255，因此0xFF + 1会导致溢出回绕到0x00。

而有符号数的0xFF表示-1，所以0xFF + 1 = 0x00就等于-1 + 1 = 0。所以0xFF + 1对于有符号数来说并没有产生异常。

下界溢出

下届溢出和上界溢出原理是一样的，也就是把加 1 变成了减 1 。也就是在汇编代码中，把add替换成了sub。

同样，下界溢出也是有两种情况：

  • 0x00 - 1 = 0xff

第一种情况仅针对无符号数有用，因为在有符号char中0x00 - 1 = -1，这个结果是正确的。但是对于无符号来说就成了 0 - 1 = 0xff，从最小值溢出回绕到了最大值。

  • 0x80 - 1 = 0x7f

第二种情况仅对于有符号数有效，因为对于无符号数来说0x80 - 1 = 0x7f，这个结果是正确的。但是对于有符号数来说就变成了-128 - 1 = 127。

整数截断

整数截断是指当将一个较大的整数类型转换为较小的整数类型时，可能会丢失数据。例如，long转换为int时，如果值超出int的范围，可能会发生截断。

整数比较

标志位操作

通过直接操作 CPU 标志位（如零标志、符号标志、溢出标志等）可能会绕过程序的逻辑，甚至在某些情况下使程序执行不当。可以使用汇编语言实现对标志位的操作，但在 C 语言中，我们通常使用内联汇编。
